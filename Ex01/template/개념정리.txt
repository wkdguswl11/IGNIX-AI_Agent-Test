Express Framework
- 프레임워크 : 개발자에게 공통적으로 개발의 틀을 제공하는 환경
    -> 장점 : 개발 속도가 향상 , 유지보수가 편하다!(폴더가 정해져있음!)
    -> 단점 : 초기에 학습하기 어려움

- Express : 기존에 서버가 하는 역할이 너무 많아서 서버가 하는 역할을 분배하는 기법(분업화)
    
    [기본구조] -> 규칠
    1) config : 설정에 관련된 파일만 관리하는 폴더! (DB연결, API key 등을 관리)
    2) public : 정적파일들을 관리하는 공간(css,js.img,video..)
    3) routes (핵심!) : 사용자의 요청에 맞게 역할을 분배하는 공간 (아이콘모양 -> 이정표)
    4) views : 동적페이지를 담당하는 공간 -> 의미는? : 페이지의 틀은 개발자가 생성! 값은 컴퓨터가 설정함!!
    5) app.js : 실질적인 서버관련 코드를 작성하는 공간 (컨트롤타워 (핵심파일!))-> 실질적인 서버

    [Express 만드는 프로세스]
    - 사용할 워크스페이스(큰 폴더 EX)왼쪽 큰 폴더) 생성 
    - Express 설치
    - 4개의 기본 폴더 생성 (직접생성!)
    - app.js 제작
